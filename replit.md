# Fumadocs Documentation Site

## Overview

This is a documentation website built with Next.js 16 and Fumadocs, a documentation framework designed for MDX-based content. The project provides a pre-configured documentation system with a home page, documentation pages rendered from MDX files, and built-in search functionality.

The site uses Fumadocs MDX for content management, allowing documentation to be written in MDX format with frontmatter support. Content is stored in the `content/docs` directory and automatically processed into navigable documentation pages.

## User Preferences

Preferred communication style: Simple, everyday language.

## System Architecture

### Frontend Framework
- **Next.js 16** with App Router architecture
- React 19 for UI components
- TypeScript for type safety throughout the codebase

### Styling
- **Tailwind CSS 4** with PostCSS integration
- Fumadocs UI preset styles (`fumadocs-ui/css/preset.css` and `fumadocs-ui/css/neutral.css`)
- Inter font from Google Fonts

### Content Management
- **Fumadocs MDX** handles MDX file processing and content loading
- Content source configuration in `source.config.ts` defines the docs collection
- The `loader()` function in `lib/source.ts` provides the interface to access content
- MDX files support frontmatter with customizable Zod schemas

### Routing Structure
- `app/[[...slug]]/page.tsx` - Catch-all route for all documentation pages (including root `/`)
- `app/[[...slug]]/layout.tsx` - DocsLayout wrapper with sidebar navigation
- Documentation is served directly from the root URL (`/`)
- Static params generation for build-time page creation

### Layout System
- Shared layout options in `lib/layout.shared.tsx` for consistent navigation
- RootProvider in `app/layout.tsx` wraps the entire app for Fumadocs UI functionality
- DocsLayout in `app/[[...slug]]/layout.tsx` provides sidebar and documentation UI

### Content Processing
- `.source/index.ts` is auto-generated by fumadocs-mdx postinstall script
- MDX components configured in `mdx-components.tsx` using Fumadocs defaults
- Lucide icons plugin integrated for icon support in docs

## Icons Gallery

The site includes an Icons section (`/icons`) with 755 SVG icons. Features:
- Grid display of all icons with search functionality
- Click any icon to copy its SVG code to clipboard
- Hover to see icon name
- Icons stored in `components/icons/` as TSX files
- Icon data extracted to `lib/icons-data.json` for client-side rendering

## External Dependencies

### Core Framework
- **Next.js 16** - React framework with App Router
- **React 19** - UI library
- **Fumadocs Core** (16.0.3) - Headless documentation utilities
- **Fumadocs UI** (16.0.3) - Pre-built documentation UI components
- **Fumadocs MDX** (13.0.1) - MDX content processing and collection management

### UI & Styling
- **Tailwind CSS 4** - Utility-first CSS framework
- **Lucide React** - Icon library

### Development Tools
- **TypeScript 5.9** - Static type checking
- **PostCSS** - CSS processing

### Build Process
- The `postinstall` script runs `fumadocs-mdx` to generate the `.source` directory
- Development server runs on port 5000 with host binding for Replit compatibility
